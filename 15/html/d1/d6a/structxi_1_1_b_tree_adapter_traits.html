Шаблон структуры xi::BTreeAdapterTraits< T >

Класс свойств (черт, traits), определяющий необходимые всопомогательные компоненты, ассоциированные с типом данных T. Подробнее...

#include <btree_adapters.h>

Открытые типы

typedef const T & TArg
 Тип неизменяемого аргумента: в большинстве случаев — конст. ссылка, но для некоторых (интегральных в частности) типов дешевле передавать по значению.
 
typedef T & TRef
 Тип-ссыка основного типа ключа.
 
typedef T TRes
 Тип результата получения ключа. В большинстве случаев по значению, так как придется выполнять преобразование типов. // TODO: Для случаев, когда дорого дважды копировать, можно рассмотреть семантику перемещения (проверить со string)
 
typedef T & TRefRes
 Тип для получение значения ключа через ссылку, передаваемую в параметр метода.
 
typedef const T * TConstPtr
 Описывает константный указатель на тип.
 

Открытые статические члены

static bool compare (const Byte *lhv, const Byte *rhv, UInt sz)
 Выполняет сравнение двух ключей дерева B-tree, как это указано в компараторе BaseBTree::IComparator. Наивная реализация приводит указатели к соответствующим указателям на типы, разыменовывает их и сравнивает без учета размера.
 
static bool isEqual (const Byte *lhv, const Byte *rhv, UInt sz)
 
static void raw2keyRes (const Byte *raw, TRef key)
 Дефолтная реализация метода преобразования потока байт в тип ключа.
 
static void key2Raw (Byte *raw, TArg key)
 Дефолтная реализация метода преобразования ключа (типом) в поток байт.

Класс свойств (черт, traits), определяющий необходимые всопомогательные компоненты, ассоциированные с типом данных T.

Введение в классы свойств см. https://accu.org/index.php/journals/442

Данные класса

template<typename T >
const UShort xi::BTreeAdapterTraits< T >::REC_SIZE = sizeof(T)
static

Размер записи, по умолчанию определяется размером типа.

Константу необходимо переопределять для сложных типов, например, string
